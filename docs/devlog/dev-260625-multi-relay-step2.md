# dev-260625-multi-relay step2

## 传输记录与锚点

- 页面重载时优先定位到上次记录锚点；没有锚点时定位到最后一条传输记录。
- 初始只渲染锚点附近记录，再后台补齐前后记录，并在补齐期间尽量保持当前滚动位置。
- 传输记录渲染增加去重和按时间插入保护，减少后台补齐与实时消息同时到达时的乱序风险。

## 磁链与缓存列表

- 修复同源 iframe 被 `X-Frame-Options: DENY` 拦截的问题，改为 `SAMEORIGIN`，使磁链缓存列表可作为浮层嵌入。
- 首页顶部下载入口改为打开磁链缓存列表浮层，适配 PWA 场景，避免跳转后无法返回首页。
- 磁链缓存列表页顶部新增磁链 ID 输入框，可直接跳转加载对应磁链下载页。

## 隧道切换与入口

- 底栏“隧道”长按弹出的切换列表中，当前隧道改为高亮样式，不再追加“当前”文字。
- 切换到其它隧道时强制重新加载对应 hash，避免仅变更地址而应用状态没有切换。
- 顶栏连续 7 次打开 admin 页改为快速连续点击，慢速累计点击会自动重置。

## 缓存与退出

- 文件清除缓存时，如果当前没有其它在线设备，会先警告确认，避免误删后无法恢复。
- 连接设备区域底部新增“退出隧道”按钮，仅清理本机当前隧道的缓存数据并离开隧道，不同步删除其它设备数据。
- 文件详情浮层禁用文本选中，修复移动端长按详情时选中文字的问题。

## 同步目录

- 同步目录新增可视化状态：选择目录、扫描目录、打包同步、同步完成、未发现变更、失败原因。
- 同步目录状态显示下次扫描倒计时，便于判断同步心跳是否仍在运行。

## 部署与日志

- Node 服务默认端口可通过 `PORT` 或 `WEB_PORT` 环境变量覆盖。
- 新增 `tunnel.config.json`，通过 `debugLogsEnabled` 控制客户端和服务端调试日志，默认关闭详细 debug log。
- Service Worker 缓存版本升级到 `instant-tunnel-v10`，并纳入 `runtime-config.js`。
