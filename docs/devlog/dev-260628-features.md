### 分片级 Attempt

## 2026-06-29 STEP2 传输完成语义收紧
### Socket.IO Relay 背压闭环
- Socket.IO relay 不再把 `socket.emit` 写入本地缓冲当成发送完成；供源端现在会等待服务端 ACK，服务端会继续等待接收端 ACK。
- relay start/chunk/complete 都形成端到端确认链：接收端处理 start 后才允许供源端继续发 chunk；每个 chunk 被接收端纳入接收缓冲后才发下一段；complete 必须等接收端组装并写入 IndexedDB 成功。
- 供源端 relay 进度条的 100% 现在表示接收端已经确认完成，而不是仅表示供源端把数据塞进 Socket.IO 缓冲区，避免“进度条消失但流量还在飙”的错觉。

### P2P 完成 ACK
- P2P DataChannel 发送端在发送 complete 后，会等待接收端返回 `file-asset-complete-ack`；接收端只有在完成组装、校验大小、写入 IndexedDB、刷新本地状态后才发送 ACK。
- 如果接收端落库失败或发现传输状态不匹配，会返回失败 ACK 或关闭通道，供源端不会提前把上传进度标为完成。
- P2P 和 Socket.IO relay 的“完成”语义现在保持一致：完成必须以接收端确认成功为准。
